The CPU's Program Counter (PC) holds the address of the next instruction. The instruction is read from Instruction Memory at that address. After the fetch, PC is automatically incremented by 4.

Project Overview
What is RISC-V?
RISC-V (Risk Five) is an open-standard Instruction Set Architecture (ISA) based on Reduced Instruction Set Computing principles, developed at UC Berkeley. Released publicly in 2010, it has become the world's fastest-growing ISA — used in everything from embedded chips to datacenters.
Why Fixed 32-bit Instructions?
Fixed-width instructions simplify hardware decoding — the CPU always knows exactly where each field is. This enables faster pipelines, simpler control units, and cleaner VLSI design compared to variable-length ISAs like x86.
Why 6 Different Formats?
Different instructions need different information. R-type needs two register sources; Load needs a base register + offset; Branch needs two comparands + a jump target. Each format packs exactly what's needed into 32 bits, nothing wasted.
5-Stage Pipeline Design
The classic 5-stage pipeline (IF→ID→EX→MEM→WB) allows 5 instructions to overlap, giving near 1 Instruction Per Clock (IPC) throughput. Hazards (data, control, structural) require stalls or forwarding — studied in advanced COA courses.

Key Design Principles of RISC-V
✅ Simplicity — Fixed-width 32-bit instructions
✅ Regularity — rs1, rs2 always at same bit positions
✅ Openness — Free, royalty-free, no licensing fees
✅ Scalability — RV32I → RV64I → RV128I extensions
✅ Modularity — M (multiply), F (float), A (atomic) extensions
✅ Clean Slate — No legacy baggage like x86 or ARM
